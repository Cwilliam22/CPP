la Class Fixed qui va être envoyé à ces opérateur. 

Dans ces fonctions
    -> 6 opérateurs de comparaison (>, <, >=, <=, ==, !=)
    -> 4 oprérateurs arithmétique (+, -, *, /)

Quelle est la logique pour avoir la bonne syntaxe lors de
l'écriture des prototypes des ces fonctions ?

Dans les exercices du module passé nous avons écris deux
choses ressemblant fortememt à ce qu'on nous demande dans
cette exercice.

    1. std::ostream& operator<<(std::ostream& os, const Fixed& f);
        std::ostream& operator<<(std::ostream& os, const Fixed& f)
        {
            os << f.toFloat();
            return os;
        }

        -> Nom de la fonction : operator<<
        -> Liée au symbole "<<"
        -> La fonction retourne une référence sur un flux de sortie (std::ostream&).
        -> Permet d'écrire ça : std::cout << a << std::endl;
        -> Retourne un flux
        -> Le "&" dans std::ostream& os, signifie qu'on ne retourne
            pas une copie du flux mais qu'on travaille sur le même flux
        -> Idem pour le "&" dans const Fixed& f

Opérateurs increment/decrement (pre-increment and post-increment, pre-decrement and post-decrement) operators

    Comment différencier la simple incrémentation de la post incrémentation ?
    Besoin d'un moyen de distinguer les deux prototypes

    (pre-increment et pre-decrement)
    (post-increment et post-decrement)
    exemple : ++a
    aucun argument dans cette fonction juste en donne +1 ou -1 à l'obj

    Logique :
        (Exemple)
        a = 5;
        ++a;

        Augmenter 5 de 1 -> devient 6
        et renvoie 6

        Ajoute ou enlève 1 avant le renvoie de la variable a

    (post-increment et post-decrement)
    exemple : a++
    Besoin d'un moyen de distinguer du prototype de pre-incrémentation
        -> int dans la fonction -> 
        -> Le int devant n’est jamais utilisé
            Il n’a pas de nom
            Il sert uniquement de marqueur syntaxique
            C'est une convention
            Le compilateur passe automatiquement zéro comme paramètre

    Logique :
        (Exemple)
        a = 5;
        a++;

        reviens à copier 5
        augmenter 5 de 1 -> devient 6
        et renvoie la copie de 5

        Ajoute ou enlève 1 après que la variable ait été renvoyé
        Autrement dit ça renvoie une ancienne valeur qui n'est plus celle de notre variable

Problemes rencontre :

    - La division et la multiplication de display pas de resultat correcte.
        Pourquoi ?
        On ne peut pas prendre les valuer brut a diviser et multiplier si ces
        derniers sont des float.
        Ce qui fonctionne '+' et '-' :
            (A * 256) + (B * 256) = (A + B) * 256
        Raison :
            (A * 256) * (B * 256) = A * B * 256 * 256 -> faut / 256
        La multiplication est associative et commutative.
            (A * 256) / (B * 256) = A / B -> faut * 256

(Pas un probleme mais une remarque sur le sujet main etc) 
    - Les 4 operateurs d'incrementation et decrementation ne renoie pas le bon resultat
        Pourquoi ?
        J ajoute +1 ou moins -1 a la variable _fixedPointValue ce qui est faux
        _fixedPointValue est un int mais n'est pas une valeur reel  (n'est pas le nombre)
        -> C’est le nombre multiplié par 256
        Par consequent mon erreur est la suivante :
        (exemple)
        _fixedPointValue = 768 (3 * 256)
        Du coup _fixedPointValue += 1 donne 767
        La valeur reel ne sera donc pas egal a 4 mais a qqch entre deux
        Il faudrait multiplier le 1 par 256 en passant pas un bit shiffting pour garder un terme generique
